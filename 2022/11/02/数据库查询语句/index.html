这是Mysql一写查询语句

发表于2022-11-02|更新于2022-11-02
|字数总计:1.8k|阅读时长:7分钟|阅读量:

简单查询：

select * from employee5；

select name,salary,sex from employee5

避免查询出现重复：

post 职位

select distinct post from employee5；

通过四则运算查询：

select name, salary, salary*14 from employee5; //查询name和salary还有14个月的salary的

select name, salary,salary14 as annul_salary from employee5 //跟上面类似只是改salary14上面的类别

定义显示格式：

concat() 函数用于连接字符串

SELECT CONCAT(name, ‘ annual salary: ‘, salary*14)  AS Annual_salary FROM employee5;

select name,concat(‘annual salary: ‘,salary*14) as Annual_salary from employee5;

单条件查询:

select post from employee5 where name=’tom’;

多条件查询****：

select name,sex,salary from employee5 where salary>3000 and salary<10000;

关键字查询：

需要用 between and

select name,salary from employee5 where salary between 3000 and 10000;


在3000到10000区间

select name,salary from employee5 where salary not between 1000 and 3000;

不在1000到3000之间的

关键字IS NULL：

select name,job_description from employee5 where job_description is not NULL;


查询job_description不为null的name和job_description

关键字IN集合查询：

select name,salary from emloyee5 where salary=5000 or salary=5500;

正常查询

select name,salary from employee5 where salary in (5000,5500);


查询工资为5000和5500的name和salary

排序查询：

order by

select id,name from employee5 order by name; //根据name的第一个字母排序

select id,name from employee5 order by name desc limit 3;//最后三个name的倒道墟

select id,name from employee5 order by name limit 1,3;//一到三的name

注：

    ascending    美音 /ə’sɛndɪŋ/   升序

    descending  美音 /dɪ’sɛndɪŋ/  降序

多列排序：

入职时间相同的人薪水不同

select * from employee5 order by hire_date desc,salary asc; //先是入职时间排序在根据入职时间相同前提条件进行薪水的升序

先按入职时间，再按薪水排序

select * from emp ORDER BY hire_date DESC,salary DESC;

先按职位，再按薪水排序

select * from employee5 ORDER BY post,salary DESC;

限制查询的记录数

SELECT * FROM employee5 ORDER BY salary DESC  LIMIT 5; //零开始总查询6个

分组查询：

group by

select dep_id,group_concat(name) as name from employee5 group by dep_id;


查询id和name根据id分组，name中的数据进行连接

子查询：

可以用的函数：count（）max（） min（）avg（）database（） user（） now（）sum（）password（）md5（） sha1（） power（）

select name from employee5 where salary(select max(salary) from employee5);


查询 工资最高的人名和工资

数据长度大小查询：

select * from employee5 where LENGTH(name) < 20; —查询name长度小于20的数据

多表查询：

内连接：

两种方式：①使用where条件 ②使用inner join

① select a.emp_id,a.emp_name,a.age,b.dept_name from employee6 a, department b where a.dept_id = b.dept_id,


查询id,name,age,dept_name数据

② select a.emp_id,a.emp_name,a.age,b.dept_name from employee6 a inner join department6 b on a.dept_id=b.dept_id;

①和②效果类似

**外连接：**

select 字段列表 from 表1 left|right join 表2 on 表1.字段=表2.字段；

左连接：left join

select emp_id,emp_name,dept_name from employee6 a left join department6 b on a.dept_id=b.dept_id;


找出所有员工及所属

的部门，包括没有部门的员工

**右连接： right join**

select emp_id,emp_name,dept_name from employee6 a right join department6 b on a.dept_id=b.dept_id;


找出所有部门包含的员工，包括空部门

全连接：full join

select * from employee6 full join department6;

select * from employee6，department6; 这个与全连接效果一样

三、复合条件连接查询

示例1：以内连接的方式查询employee6和department6表，并且employee6表中的age字段值必须大于25

找出公司所有部门中年龄大于25岁的员工

select emp_id,emp_name,age,dept_name FROM employee6,department6 WHERE employee6.dept_id=department6.dept_id AND age > 25;

示例2：以内连接的方式查询employee6和department6表，并且以age字段的升序方式显示

SELECT emp_id,emp_name,age,dept_name FROM employee6,department6 where employee6.dept_id=department6.dept_id ORDER BY age asc;

四、子查询

子查询是将一个查询语句嵌套在另一个查询语句中。

内层查询语句的查询结果，可以为外层查询语句提供查询条件。

子查询中可以包含：IN、NOT IN、ANY、ALL、EXISTS 和 NOT EXISTS等关键字

还可以包含比较运算符：= 、 !=、> 、<等

1. 带IN关键字的子查询

查询employee表，但dept_id必须在department表中出现过

SELECT * FROM employee6 WHERE dept_id IN (SELECT dept_id FROM department6);

2. 带比较运算符的子查询

=、!=、>、>=、<、<=、<>

查询年龄大于等于25岁员工所在部门（查询老龄化的部门）

SELECT dept_id,dept_name FROM department6 WHERE dept_id IN (SELECT DISTINCT dept_id FROM employee6 WHERER age >= 25);

3. 带EXISTS关键字的子查询

EXISTS关字键字表示存在。在使用EXISTS关键字时，内层查询语句不返回查询的记录，而是返回一个真假值。

True或False，当返回True时，外层查询语句将进行查询；当返回值为False时，外层查询语句不进行查询

department表中存在dept_id=203，Ture

SELECT * from employee6 WHERE EXISTS (SELECT * FROM depratment6 WHERE dept_id=203);

4、对比两个表：

select  *  from  bm_facility_info_copy1   A  where  NOT EXISTS (select 1 from bm_fwz_info_copy1  B  where  A.fwzCode = B.fwzCode);

5、通过一张表的字段数据查另一张表

select *  from bm_fwz_info_copy3 where fwzCode in (select fwzCode from bm_fwz_info_copy5)

//copy3是要查询的表 、copy5是对比的表

6、将查询结果插入另一张表

insert into bm_fwz_infoxm select * from bm_fwz_info A where not  EXISTS(select fwzCode from bm_facility_infoxm B where B.fwzCode=A.fwzCode)

7、将查询结果从另一张表删除

delete from bm_fwz_infoxm  where fwzCode in (select fwzCode from bm_fwz_info A where not  EXISTS(select fwzCode from bm_facility_infoxm B where B.fwzCode=A.fwzCode))

8、查询某字段在哪张表里

SELECT * from information_schema.columns where TABLE_SCHEMA=’数据库’ and COLUMN_NAME=’某字段名’

9、****查询某段时间里某个字段里数据出现的个数（Group by）

select type,count(*)  from 表 where create_time >’2021-01-01 00:00:00’ AND create_time < ‘2022-01-01 00:00:00’ group by type

10、模糊查询数据并取反

SELECT * FROM bm_fwz_info WHERE fwzCode NOT IN (SELECT fwzCode FROM bm_fwz_info WHERE fwzCode LIKE “%nd%”);

11、查询同一张表下的数据并删除

delete from code_user where user_id in (select user_id from (select user_id from code_user where language = “java”) as tmp);

delete from bm_fwz_info_copy1 where fwzCode in (select fwzcode from (select * from bm_fwz_info_copy1 where fwzCode not in (select fwzCode from bm_fwz_info_copy1 where fwzAddress like “%厦门%”))  AS t)

注：select fwzcode from 关键 输出一个临时表

12、将查询结果创建为一张表

create table zxw (select * from bm_fwz_info_copy1 where fwzCode not in (select fwzCode from bm_fwz_info_copy1 where fwzAddress like “%厦门%”))
